Given an array with n objects colored red, white or blue, sort them so that objects of the same color are adjacent, with the colors in the order red, white and blue.

Here, we will use the integers 0, 1, and 2 to represent the color red, white, and blue respectively.

Note:
You are not suppose to use the library's sort function for this problem.

click to show follow up.

Follow up:
A rather straight forward solution is a two-pass algorithm using counting sort.
First, iterate the array counting number of 0's, 1's, and 2's, then overwrite array with total number of 0's, then 1's and followed by 2's.

Could you come up with an one-pass algorithm using only constant space?

一开始使用了快排加插入的方式 nlogn
public class Solution {
    public void sortColors (int[] A) {
        quickSort (A, 0 , A. length);
    }

    public void quickSort (int[] A, int s, int e) {
         if ( e - s <= 7) {
            insertSort (A, s, e);
             return;
         }
         int posi = partition(A , s, e - 1);
        quickSort (A, s, posi);
        quickSort (A, posi + 1, e );
    }

    public int partition (int[] A, int p, int q) {
         int pivot = A[p ];
         while ( p < q) {
             while ( A[q] > pivot && p < q ) {
                q --;
             }
            A [p] = A[q ];
             while ( A[p] < pivot && p < q ) {
                p ++;
             }
            A [q] = A[p ];
            q --;
         }
        A [p] = pivot;
         return p;
    }

    public void insertSort (int[] A, int s, int e) {
         if ( null == A || e - s <= 1 )
             return;
         for ( int i = s; i < e ; i++) {
             int j = 1 ;
             int pivot = A[i ];
             while ( j <= i && pivot < A[ i - j]) {
                A [i - j + 1] = A[ i - j];
                j ++;
             }
            A [i - j + 1] = pivot;
         }
    }
}

一遍循环

I can describe it in this way: - p is the input iterator, reads new values, is as faster than others so no problems of reading values already overwritten. - i counts the number of 0 found, and a the same time writes zeroes. - j counts the number of 0 + 1 found, and at the same time writes 1. - k counts the number of 0 + 1 + 2 found and at the same time writes 2.

Your solution can be improved in this way:
